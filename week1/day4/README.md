📅 Day 4: Relational Joins & Nested Subqueries
Mastered the ability to link multiple tables and perform Subqueries (queries inside queries). 
This is critical for answering complex questions where the answer depends on a dynamic value (like an average or a maximum).
📝 Concepts CoveredOuter Joins:
Using LEFT JOIN to ensure no data is lost, even if a department has no employees. 
Subqueries:
Using SELECT statements inside a WHERE clause to filter data against calculated values. 
Correlated Subqueries:
Comparing a row’s value against its specific category average (e.g., comparing an employee's salary to their specific department's average).
