Day 4: Data Sorting & Multi-Table Joins
🎯 Goal
The focus today was on Data Presentation and Relational Connectivity.
I learned how to organize large datasets for readability and how to link disparate tables using Primary and Foreign Keys.
📝 Concepts CoveredMulti-Level Sorting: 
Ordering data by one column and then another (e.g., Department first, then Salary). 
Internal Joins (JOIN):
Matching records between the Employee, Department, and Project tables.
Left Outer Joins:
Learning how to preserve data from one table even if there is no match in the other (e.g., showing a Department even if it currently has no assigned Employees).
💻 Practice HighlightsDepartmental Expenditure Mapping:
Combined SUM() with ORDER BY to rank departments by their total salary cost.
Project Assignment Tracking:
Created a view that links specific employee names directly to the project names managed by their departments.
Full Organizational Audit:
Used LEFT JOIN to identify gaps in the organization, such as departments without staff or employees without assigned departments.
